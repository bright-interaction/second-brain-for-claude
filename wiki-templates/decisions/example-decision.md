# Example decision

A decision note records a choice and the reasoning, so the next person (or AI) does
not relitigate it. One decision per note. Keep the flywheel fields tight.

**Do:** capture the decision the moment it is made, with `mesh_append_note type=decision`.

**Don't:** bury the reasoning inside an entity page or a chat log where search can't tier-0-boost it.

**Why:** Mesh ranks `decision`/`gotcha`/`post-mortem` notes first and reserves part of
every search budget for them, so a discrete decision note is the highest-value thing
you can write. Delete this example once you have a real one.
