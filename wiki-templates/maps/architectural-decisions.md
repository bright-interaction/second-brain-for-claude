# Architectural Decisions

ADR-style record of choices that shape this project. Each entry has Decision, Why, Trade-off, Status. Reverse chronological order: newest at the top. Append new entries with the `/adr` slash command.

ADR numbers are sequential and never reused, even when an entry is later reverted.

**Decision.** Replace this entry with a real architectural decision. The `/adr` slash command will insert new ones above this line, so you can leave this as a working example until you have a real second ADR to point to.
